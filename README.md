# Kuber Timesheet

A cloud-based attendance tracking web application designed for offices with secure authentication, Nepali calendar support, and comprehensive audit tracking. Built for audit firms with client tracking capabilities.

## 🎯 Features

### 🔐 Authentication & Access Control
- Email + Password login (pre-generated by admin)
- Only pre-approved email addresses can access
- No public registration/signup
- Role-based access: Admin & Employee

### 👤 Employee Features
- **Clock In/Out** — Record work hours with automatic time rounding
- **View Records** — Monthly attendance records in Nepali calendar
- **Client Tracking** — Manual client name entry for audit purposes
- **Edit Restrictions** — Can only edit entries same-day before 12:00 PM
- **Personal Dashboard** — Your attendance summary and quick actions

### 🛠️ Admin Features
- **Employee Management** — Add, edit, deactivate employees
- **Holiday Management** — Define holidays for the organization
- **Full Record Access** — View and edit any employee's records anytime
- **Audit Dashboard** — Monitor all attendance changes
- **Excel Export** — Download attendance data with filters

### ⏱️ Time & Attendance
- **Smart Time Rounding** — Rounds to nearest 5 or 0 minutes (9:02 → 9:05, 9:56 → 10:00)
- **Auto Calculation** — Hours worked = Time Out − Time In
- **Nepali Calendar Support** — All dates stored in Bikram Sambat (BS) format
- **Nepal Time (NPT)** — UTC+5:45 timezone handling
- **Lock System** — Entries locked after 12:00 PM same-day (employees only)

### 📊 Reporting
- **Monthly Summaries** — Present days, Leave days, Hours worked, Holidays
- **Attendance Tracking** — Auto-count leave when no entry exists
- **Holiday Exclusion** — Holidays not counted as absences
- **Excel Export** — Filterable exports by employee, date range

### 🔒 Security
- **Row-Level Security (RLS)** — Database-level access control
- **Audit Logging** — All changes tracked with timestamps
- **Session Management** — Secure JWT-based authentication
- **Data Isolation** — Employees see only their own records

## 🚀 Quick Start

### Prerequisites
- Node.js v18+
- Python 3.9+ (via uv)
- Supabase Account
- Vercel Account (for deployment)

### Local Setup

1. **Clone and Setup:**
   ```bash
   cd frontend
   npm install
   ```

2. **Configure Environment:**
   - Copy `.env.local` template
   - Add Supabase credentials (URL, Anon Key, Service Role Key)
   - Set Nepali API URL: `http://localhost:8000`

3. **Setup Python Microservice:**
   ```bash
   cd services/nepali-date-service
   uv pip install -r requirements.txt
   python app.py
   ```
   - Service runs on `http://localhost:8000`
   - Swagger docs: `http://localhost:8000/docs`

4. **Run Frontend:**
   ```bash
   cd frontend
   npm run dev
   ```
   - Open `http://localhost:3000`

### Deployment

See [SETUP_GUIDE.md](./SETUP_GUIDE.md) for complete deployment instructions:
- Supabase database setup
- Vercel frontend deployment
- Python microservice serverless deployment

## 🗄️ Database Schema

### Users Table
- Email, Name, Role (admin/employee), Is Active

### Attendance Records
- User ID, Entry Date (Nepali), Time In, Time Out, Hours Worked (auto), Client Name, Notes, Is Locked

### Holidays
- Holiday Date (Nepali), Description

### Audit Logs
- User ID, Action (clock_in/clock_out/edit/delete), Changes (old/new values), Timestamp

## 🔗 API Endpoints

### Authentication
- `POST /api/auth/login` — Login with credentials
- `POST /api/auth/logout` — Sign out

### Attendance
- `POST /api/attendance` — Clock in (with time rounding)
- `PUT /api/attendance` — Clock out (with hours calculation)
- `GET /api/records` — Get user's attendance records (filterable by month)

### Holidays (Admin)
- `GET /api/holidays` — List all holidays
- `POST /api/holidays` — Create holiday

### Nepali Date Conversion (Python Microservice)
- `POST /convert/english-to-nepali` — Convert Gregorian to Bikram Sambat
- `POST /convert/nepali-to-english` — Convert Bikram Sambat to Gregorian
- `GET /date/current` — Current date in both calendars (NPT timezone)

## 🐛 Troubleshooting

**Can't login?**
- Ensure Supabase database schema is created (`database_schema.sql`)
- Verify user exists in `auth.users` table
- Check user exists in `users` table with correct role

**Time not rounding?**
- Verify Python microservice is running
- Check `NEXT_PUBLIC_NEPALI_API_URL` is correct
- Test microservice: `curl http://localhost:8000/health`

**Nepali dates not showing?**
- Ensure Python dependencies are installed: `uv pip install -r requirements.txt`
- Restart Python service: `python app.py`
